# 电商平台融资自动巡检风险规则清单

本文是 `产品文档.md` 的风险规则落地补充，产品定位、功能边界、建议动作、规则引擎与 Agent 分工、权限和留痕要求均以 `产品文档.md` 为准。本文只定义风险点、主规则关系、金额处理口径、证据要求、首期优先级和二期/治理边界。

## 一、继承的产品边界

### 1.1 统一边界

本系统只负责发现风险、解释风险、通知相关人员和留痕，不决定是否放款，也不在系统内执行放款、停放、调额、催收或法务动作。最终放款判断和业务动作仍在原业务系统和原业务流程中完成。

### 1.2 规则引擎与 Agent 边界

规则引擎是风险事件、风险等级、影响金额、建议动作、通知策略和证据的唯一来源。Agent 只负责把规则引擎输出写成通知摘要、完整报告、风险解释和核实事项，不得新增事实、改写等级、改写金额、改写建议动作或补齐缺失字段。

### 1.3 建议动作边界

本文使用的建议动作枚举继承产品文档：

- `observe`：正常观察。
- `cautious`：谨慎放款。
- `reduce_ratio`：建议降低参考比例。
- `pause_new`：建议暂停新增。
- `manual_check`：人工核实。

所有建议动作都是报告输出建议，不代表系统已经执行对应业务动作。

## 二、风险规则总原则

### 2.1 第一还款来源

本产品的第一还款来源是订单形成的净可归集货款。GMV、订单量、主体信用、法人负债、经营利润、达人依赖、行业政策等可以作为风险信号，但不能直接等同于可回款资产。

### 2.2 什么可以影响金额

只有已经落到订单、售后、履约、结算、平台扣款、回款账户、权属或扣收优先级证据上的风险，才能影响可回款资产金额。

辅助风险不能直接扣减金额。辅助风险若要影响金额，必须转化为某个主规则的证据，例如：

- 退款转入 R16 售后退款风险。
- 平台扣款转入 R18 平台扣款风险。
- 回款账户异常转入 R20 回款路径控制风险。
- 权属冲突转入 R21 应收款权属与重复融资风险。
- 平台处罚转入 R05 平台违规风险。

### 2.3 同一金额只扣一次

同一笔金额只能由一个主规则形成扣减项、冻结项、打折项或剔除项。

广告费、达人佣金、平台佣金、技术服务费、违约金、赔付、官方金融扣款等优先扣款统一由 R18 平台扣款风险计算。R23、R25、R27 等风险点只能引用 R18 的扣减结果，不再重复扣减。

### 2.4 数据缺失不等于业务风险事实

数据缺失、授权失效、接口延迟、字段缺失或口径冲突，不等于确认发生业务风险。系统应先输出“数据不足”或“无法估算原因”。只有存在账户变更、代扣授权失效、回款未进入受控路径等业务证据时，才能进一步标记回款失控。

### 2.5 订单入池、打折、冻结和剔除

订单不是支付成功后就自动进入可回款资产。只有在订单真实性、履约状态、售后状态、预计结算日、预计扣款项、回款路径和权属状态满足配置口径时，才可计入可回款资产。

规则处理方式包括：

| 处理方式 | 适用场景 |
| --- | --- |
| 正常计入 | 订单真实、履约稳定、售后风险低、结算和回款路径清晰 |
| 打折计入 | 存在可量化风险，但仍有较高概率形成回款 |
| 形成扣减项 | 已发生退款、取消、赔付、罚款、佣金、广告费、保证金补缴等 |
| 形成风险冻结项 | 货款冻结、结算暂停、账户异常、司法协查等 |
| 剔除 | 订单取消、退款成功、虚假交易、权属不清、无法扣收或证据不足 |
| 无法判断 | 关键字段缺失、接口延迟、授权失效或口径冲突 |

### 2.6 风险结果和影响类型

本文统一使用以下影响类型：

| 影响类型 | 含义 | 示例 |
| --- | --- | --- |
| 金额缩水 | 可回款资产金额减少 | 退款、赔付、罚款、佣金扣除、广告费扣除、保证金补缴 |
| 到账延迟 | 可回款资产到账晚于预计时间 | 账期延长、物流异常、售后未完结 |
| 风险冻结 | 资产存在但暂不能归集或扣收 | 货款冻结、结算暂停、账户异常金额 |
| 资产灭失 | 单笔订单或资产不再成立 | 订单取消、退款成功、虚假交易、平台否定交易 |
| 回款失控 | 回款路径或扣收路径不稳定 | 结算账户变更、监管账户失效、代扣授权失效、资金被快速转走 |
| 权属/优先级争议 | 资产归属或扣收优先级不清 | 重复融资、重复质押、司法冻结、其他机构优先扣收 |
| 数据不足 | 关键字段缺失或不可用 | 授权失效、接口延迟、字段缺失、口径冲突 |
| 辅助风险 | 不直接扣减资产但影响判断 | 主体异常、法人失信、经营下滑、利润恶化、多头负债 |

货款冻结需要区分类型：

| 冻结类型 | 归类方式 |
| --- | --- |
| 电商平台临时风控冻结 | 通常归为风险冻结，报告中同时说明可能导致到账延迟 |
| 违规处罚或赔付相关冻结 | 可能同时形成风险冻结和金额缩水 |
| 司法冻结或债权争议冻结 | 通常归为权属/优先级争议，严重时形成回款失控 |

## 三、产品对象和数据来源边界

### 3.1 首期稳定对象

以下对象可以进入首期自动规则：

| 对象 | 示例 |
| --- | --- |
| 商家 | 企业主体、融资合同、店铺经营主体 |
| 店铺 | 店铺状态、平台权限、体验分或服务评分、API 授权 |
| 商品/SKU | 商品状态、类目、资质、生命周期 |
| 订单 | 订单状态、金额、支付、取消、退款、履约 |
| 售后单 | 退款、退货、纠纷、平台介入 |
| 物流单 | 发货、揽收、签收、拒收、异常轨迹 |
| 结算单 | 账期、待结算金额、扣款、冻结、到账日 |
| 回款账户 | 监管账户、结算账户、代扣授权、账户冻结 |
| 系统配置 | 规则版本、资产口径版本、通知策略、权限配置 |

### 3.2 条件保留或二期对象

以下对象只有在数据源稳定、字段口径明确后才能进入自动规则；否则只能作为人工备注、辅助标记或二期附录：

| 对象 | 示例 | 默认处理 |
| --- | --- | --- |
| 供应商 | 供应商断供、采购合同、供应商资质 | 二期或转入履约风险证据 |
| 库存/仓库 | 库存数量、锁库、仓库履约能力 | 有数据时条件保留 |
| 渠道对象 | 达人、直播间、短视频、广告计划 | 平台适配或二期 |
| 行业/区域 | 行业政策、区域管控、自然灾害 | 外部观察或附录 |
| 系统治理对象 | 规则、口径、模板、提示词、用户权限 | 系统治理事件，不作为商户资产风险 |

### 3.3 数据源进入规则的最低要求

风险点进入自动规则前，必须满足：

1. 有稳定数据源。
2. 能落到产品对象。
3. 能提供证据摘要和数据时间。
4. 能说明是否影响金额。
5. 能输出建议动作或无法判断原因。
6. 能记录规则版本和资产口径版本。

没有明确数据源的风险点不得进入首期自动规则。

## 六、逐项风险规则

### R01 企业工商风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则、权属优先级引用规则。
- 关联对象：商家主体、融资合同、结算账户。
- 触发信号：经营异常、严重违法失信、注销或吊销、注册地址失联、股权冻结或出质、重大行政处罚、重大司法案件。
- 影响口径：主体异常不直接扣减单笔订单资产；当影响合同主体、店铺认证、结算账户、司法冻结或扣收优先级时，转入 R20 回款路径控制风险或 R21 应收款权属与重复融资风险。
- 建议动作：cautious、manual_check；有合同主体或账户控制异常证据时可引用 R20/R21 输出 pause_new。
- 核实事项：核实主体状态、合同主体、店铺主体、收款主体和回款账户是否一致。

### R02 法人及实控人风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则、账户路径引用规则。
- 关联对象：法人、实控人、关联店铺、回款账户。
- 触发信号：法人失信、限制高消费、重大负债、实控人失联、法人频繁变更、法人和实际运营人不一致、关联店铺被处罚或清退。
- 影响口径：不作为第一还款来源扣减项；当导致授权中断、资料无法补充、账户被司法冻结、商家失联或账户控制失效时，转入 R20。
- 建议动作：cautious、manual_check；存在账户控制或授权失效证据时引用 R20 输出 pause_new。
- 核实事项：核实法人、实控人、运营联系人、授权续期、账户控制和关联店铺风险。

### R03 店铺状态风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则、结算延迟规则、账户路径引用规则。
- 关联对象：店铺、商品、订单、结算账户。
- 触发信号：店铺封禁、停业整顿、限制提现、限制发布商品、限制上新、主体认证失效、清退通知。
- 影响口径：限制经营影响新增订单和履约；限制提现、清退、主体认证失效等会影响归集和扣收。相关订单按证据打折、冻结或剔除。结算账户变更和代扣授权失效统一转入 R20；API 授权失效统一转入 R30。
- 建议动作：reduce_ratio、manual_check；限制提现、清退或账户路径异常时引用 R20 输出 pause_new。
- 核实事项：核实店铺当前状态、处罚原因、可恢复时间、账户变更原因和平台通知。

### R04 店铺体验分与服务能力风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则、结算延迟引用规则。
- 关联对象：店铺、订单、售后单。
- 触发信号：体验分或服务评分快速下降，差评、投诉、纠纷、平台介入和售后率上升。
- 影响口径：体验分下降不直接扣减资产；当导致售后退款时转入 R16，当导致履约或物流异常时转入 R15/R12，当导致平台限制或处罚时转入 R05，当导致账期变化时转入 R17。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实评分下降原因、售后集中商品、平台限制和近期账期变化。

### R05 平台违规风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则、结算延迟规则、风险冻结规则。
- 关联对象：店铺、商品、订单、结算单。
- 触发信号：虚假宣传、违规经营、售假、违规营销、平台处罚、扣罚违约金、货款冻结、清退通知。
- 影响口径：平台罚款、赔付和违约金形成扣减项；临时风控冻结形成风险冻结项；清退或交易否定可导致相关资产剔除。商品资质本身归 R07，直播或短视频内容证据归 R26，最终处罚结果归 R05。
- 建议动作：reduce_ratio、manual_check；清退、严重冻结或大额扣罚时输出 pause_new。
- 核实事项：核实处罚类型、处罚金额、冻结金额、申诉状态、预计解冻或扣款时间。

### R06 虚假发货与欺诈发货风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则、结算延迟规则。
- 关联对象：订单、物流单、商品。
- 触发信号：无揽收、空包、物流轨迹异常、批量虚假签收、收货地址异常集中、平台判定欺诈发货。
- 影响口径：平台否定交易、退款成功或处罚的订单剔除或扣减；仅物流异常但仍可履约的订单按延迟或折扣处理。普通延迟发货走 R15，仓配或承运商能力问题走 R12。
- 建议动作：reduce_ratio、manual_check；平台判定欺诈或批量异常时输出 pause_new。
- 核实事项：核实物流凭证、发货仓、买家签收、平台判责和对应订单金额。

### R07 商品合规风险

- 落地层级：条件保留。
- 规则类型：资产扣减规则、辅助预警规则。
- 关联对象：商品、SKU、订单。
- 触发信号：禁售、限售、资质缺失、虚假宣传、质量抽检不合格、售假或侵权。
- 影响口径：商品下架本身不必然导致已履约订单灭失；需按订单状态判断是否仍可结算。对应退款转入 R16，平台处罚或罚款转入 R05/R18，无法履约订单转入 R15。
- 建议动作：cautious、manual_check；有明确处罚、无法履约或大额受影响订单时引用主规则输出 reduce_ratio 或 pause_new。
- 核实事项：核实商品资质、下架原因、受影响订单、售后率和平台处罚。

### R08 类目风险

- 落地层级：条件保留。
- 规则类型：辅助预警规则、结算延迟引用规则。
- 关联对象：类目、商品、店铺资质。
- 触发信号：类目准入门槛变化、资质到期、类目权限被取消、高退货或高投诉类目占比上升。
- 影响口径：类目风险主要影响新增订单、履约能力和平台监管强度，不直接扣减资产。影响具体商品、订单或结算时，转入 R07、R15、R16 或 R17。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实类目资质、平台规则变化、受影响 SKU、订单和售后。

### R09 商品生命周期风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则、资产扣减引用规则。
- 关联对象：商品、SKU、订单。
- 触发信号：核心商品下架、销量快速衰退、库存清仓、季节性商品过季、商品评价恶化。
- 影响口径：主要影响未来订单和在途订单履约；已发货且可结算订单不直接剔除。未发货或无法履约订单转入 R15，库存不足转入 R11。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实核心商品销售占比、下架原因、库存、履约状态和售后趋势。

### R10 供应商风险

- 落地层级：首期辅助标记或二期。
- 规则类型：辅助预警规则、履约引用规则。
- 关联对象：供应商、商品、库存、订单。
- 触发信号：供应商断供、涨价、纠纷、资质失效、账期收紧、核心供应商集中度过高。
- 影响口径：供应商风险不直接扣减已确认资产；当导致缺货、无法发货、退款或赔付时，转入 R11、R15 或 R16。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实供应商履约、采购合同、库存覆盖、替代供应商和受影响订单。

### R11 库存风险

- 落地层级：条件保留。
- 规则类型：资产扣减规则、结算延迟规则。
- 关联对象：SKU、库存、仓库、订单。
- 触发信号：库存不足、超卖、库存账实不符、库存周转异常、临期或滞销库存占比高。
- 影响口径：有稳定库存数据时，未发货订单若库存不足，应按无法履约概率打折或剔除；已签收且售后风险低的订单不因库存不足重复扣减。没有库存数据时，只能作为 R15 订单履约风险的人工核实事项。
- 建议动作：reduce_ratio、manual_check。
- 核实事项：核实库存数量、锁库情况、采购到货时间、受影响订单和退款预期。

### R12 仓配物流风险

- 落地层级：条件保留。
- 规则类型：结算延迟规则、资产扣减引用规则。
- 关联对象：仓配对象、物流单、订单。
- 触发信号：揽收延迟、配送停滞、拒收率上升、区域物流中断、仓库发货能力下降。
- 影响口径：主要形成到账延迟；若导致订单取消、退款、赔付或平台处罚，再转入 R15、R16 或 R05。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实异常物流单、承运商、区域、预计恢复时间和售后影响。

### R13 订单真实性风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则。
- 关联对象：订单、买家、支付、收货地址。
- 触发信号：刷单、虚假交易、异常买家集中、异常地址集中、订单金额异常、支付与收货行为异常、平台判定交易无效。
- 影响口径：虚假或无效订单不按正常口径计入可回款资产；平台判定无效、取消或退款成功的订单剔除。数据缺失只标记数据不足，不作为订单真实性风险本身。
- 建议动作：reduce_ratio、manual_check；批量虚假交易时输出 pause_new。
- 核实事项：核实订单证据、买家分布、支付链路、物流签收和平台判责。

同一真实订单被重复融资不属于订单真实性风险，应纳入 R21 应收款权属与重复融资风险。

### R14 订单集中度风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则。
- 关联对象：订单、买家、商品、渠道、区域。
- 触发信号：订单集中在少数买家、少数商品、少数达人、少数直播间、少数区域或短时间异常爆发。
- 影响口径：订单集中度是全局风险因子，不单独作为资产扣减项；当集中对象出现退款、拒收、渠道异常或虚假交易证据时，再转入 R13、R15 或 R16。
- 建议动作：cautious、manual_check；只有转入主规则后才输出 reduce_ratio。
- 核实事项：核实集中对象、交易背景、历史基准、退款拒收和渠道真实性。

### R15 订单履约风险

- 落地层级：首期核心自动规则。
- 规则类型：结算延迟规则、资产扣减规则。
- 关联对象：订单、物流单、售后单。
- 触发信号：未发货、超时发货、未揽收、拒收、长时间未签收、履约状态停滞。
- 影响口径：未完成履约的订单通常不能按正常口径全额计入；延迟但仍可履约的订单后移预计结算日，无法履约的订单扣减或剔除。
- 建议动作：reduce_ratio、manual_check。
- 核实事项：核实订单状态、发货凭证、签收情况、预计结算日和售后风险。

### R16 售后退款风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则。
- 关联对象：售后单、订单、商品。
- 触发信号：退款率上升、退货退款成功、平台介入、仅退款增加、纠纷订单增加、大额售后集中。
- 影响口径：已成功退款、预计退款和售后扣减形成扣减项；售后未完结的订单按风险比例打折或形成风险冻结项；退款成功的单笔订单视为资产灭失。
- 建议动作：reduce_ratio、manual_check；批量退款或核心商品售后异常时输出 pause_new。
- 核实事项：核实售后原因、金额、状态、平台判责、是否可二次销售和受影响订单。

### R17 结算周期风险

- 落地层级：首期核心自动规则。
- 规则类型：结算延迟规则、风险冻结规则。
- 关联对象：结算单、订单、店铺。
- 触发信号：账期延长、预计结算日后移、结算暂停、临时风控冻结、司法协查冻结。
- 影响口径：账期延长后移预计到账日；临时冻结形成风险冻结项；司法冻结需转入 R21 权属/优先级争议判断。
- 建议动作：cautious、reduce_ratio、manual_check；结算暂停或冻结金额覆盖敞口时输出 pause_new。
- 核实事项：核实结算规则、冻结类型、冻结金额、预计解冻时间和平台通知。

### R18 平台扣款风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则。
- 关联对象：结算单、费用项、订单。
- 触发信号：平台佣金、技术服务费、达人分销佣金、违约金、赔付、广告费、官方金融扣款等优先扣除。
- 影响口径：已发生或可预计的平台扣款形成扣减项。R18 是平台费用和优先扣款的唯一金额扣减入口，R23、R25、R27 只能引用 R18，不得重复扣减。
- 建议动作：reduce_ratio、manual_check。
- 核实事项：核实扣款类型、金额、优先级、扣款日期、是否可申诉和是否已在其他费用项计算。

### R19 保证金风险

- 落地层级：首期核心自动规则。
- 规则类型：资产扣减规则、结算延迟规则。
- 关联对象：店铺、保证金、结算单。
- 触发信号：保证金不足、保证金补缴、保证金被扣、保证金类目要求提高、保证金未补导致权限受限。
- 影响口径：保证金补缴或扣罚形成扣减项；未补缴导致店铺限制时，转入 R03 或 R17。若保证金扣款已经作为平台扣款进入 R18，不得在 R19 重复扣减。
- 建议动作：reduce_ratio、manual_check。
- 核实事项：核实保证金余额、补缴要求、扣罚原因、补缴期限和权限影响。

### R20 回款路径控制风险

- 落地层级：首期核心自动规则。
- 规则类型：账户路径规则、风险冻结规则。
- 关联对象：回款账户、监管账户、代扣授权、结算账户。
- 触发信号：结算账户变更、监管账户失效、商家解绑银行卡、代扣授权失效、账户冻结、回款未进入受控路径。
- 影响口径：应收款存在不等于融资方能优先拿到钱。回款路径失控时，对应资产应形成风险冻结项、回款失控标记或剔除。仅数据不可见时先走 R30，不直接判定回款失控。
- 建议动作：manual_check、pause_new。
- 核实事项：核实合同主体、店铺主体、收款主体、放款主体、应收款权属、结算账户、代扣授权和监管路径是否一致且有效。

### R21 应收款权属与重复融资风险

- 落地层级：首期核心自动规则。
- 规则类型：权属优先级规则、风险冻结规则。
- 关联对象：订单、应收账款、融资合同、质押登记、回款账户。
- 触发信号：同一订单重复融资、同一应收款重复质押、其他金融机构优先扣收、供应商主张货权、司法冻结、主体错配。
- 影响口径：不判断订单是否真实，而判断融资方是否具备优先回收权；权属不清的资产不按正常口径计入，必要时形成权属争议冻结项。首期只处理基础权属和重复融资；复杂供应链确权放二期。
- 建议动作：manual_check、pause_new。
- 核实事项：核实合同、确权、质押登记、平台扣款优先级、其他融资记录和司法限制。

### R22 经营趋势风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则。
- 关联对象：店铺、订单、GMV、买家。
- 触发信号：GMV 下滑、订单量下降、客单价异常、复购下降、退款率上升、新客减少。
- 影响口径：经营趋势不直接等同于可回款资产；用于判断后续订单池稳定性和放款参考关注度，不单独扣减已确认资产。若趋势变化已经表现为履约或售后问题，转入 R15 或 R16。
- 建议动作：observe、cautious、manual_check。
- 核实事项：核实趋势变化原因、活动周期、核心商品、渠道变化和售后情况。

### R23 利润质量风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则。
- 关联对象：店铺、费用项、订单。
- 触发信号：毛利下降、退货成本上升、佣金和广告费过高、补贴过高、经营利润长期为负。
- 影响口径：利润质量是辅助风险，不直接扣减资产。可结算前优先扣除的费用统一归入 R18；退款成本转入 R16；广告投流效率问题转入 R27 作为辅助标记。
- 建议动作：cautious、manual_check；只有引用 R18/R16 后才输出 reduce_ratio。
- 核实事项：核实成本、佣金、广告费、退款成本、平台扣款和可回款资产覆盖敞口情况。

### R24 多头融资与负债风险

- 落地层级：首期辅助标记。
- 规则类型：辅助预警规则、权属优先级引用规则。
- 关联对象：商家、法人、融资合同、应收账款。
- 触发信号：多家机构融资、平台官方金融优先扣款、外部借款压力高、同一资产多次融资。
- 影响口径：普通负债属于辅助风险；同一应收款重复融资或优先扣收冲突转入 R21，账户路径冲突转入 R20。
- 建议动作：cautious、manual_check；出现同一资产重复融资证据时引用 R21 输出 pause_new。
- 核实事项：核实融资记录、还款安排、优先扣款协议、应收款确权和资产是否重复使用。

### R25 达人带货风险

- 落地层级：二期或平台适配。
- 规则类型：辅助预警规则、平台扣款引用规则。
- 关联对象：达人、直播间、订单、佣金。
- 触发信号：订单高度依赖单一达人、达人违规、佣金异常、达人退款率高、渠道订单异常集中。
- 影响口径：达人依赖和达人违规本身不直接扣减资产。达人佣金按 R18 处理；订单真实性异常转入 R13；售后异常转入 R16。
- 建议动作：cautious、manual_check；只有引用主规则后才输出 reduce_ratio。
- 核实事项：核实达人合作状态、佣金比例、订单质量、退款率和平台处罚。

### R26 直播间与短视频内容风险

- 落地层级：二期或平台适配。
- 规则类型：辅助预警规则、平台处罚引用规则。
- 关联对象：直播间、短视频、商品、订单。
- 触发信号：直播违规、虚假宣传、内容下架、流量限制、投诉和平台处罚。
- 影响口径：内容违规本身不直接扣减资产。平台处罚转入 R05，商品资质问题转入 R07，退款赔付转入 R16/R18，账号限制或店铺限制转入 R03。
- 建议动作：cautious、manual_check；只有引用主规则后才输出 reduce_ratio 或 pause_new。
- 核实事项：核实违规内容、处罚范围、受影响商品、订单和售后。

### R27 广告投流风险

- 落地层级：二期或平台适配。
- 规则类型：辅助预警规则、平台扣款引用规则。
- 关联对象：广告计划、费用项、订单。
- 触发信号：广告消耗快速上升、ROI 下降、投流费用优先扣款、投流带来低质量订单。
- 影响口径：投流效率下降作为辅助风险，不直接扣减资产。已发生或可预计的广告费优先扣款统一转入 R18；投流带来低质量订单时转入 R13 或 R16。
- 建议动作：cautious、manual_check；只有引用主规则后才输出 reduce_ratio。
- 核实事项：核实广告费扣款、ROI、投流订单退款率、投放计划和费用优先级。

### R28 行业政策风险

- 落地层级：二期或附录。
- 规则类型：辅助预警规则。
- 关联对象：行业、类目、商品。
- 触发信号：监管政策变化、平台规则升级、行业整治、禁限售范围变化。
- 影响口径：行业政策作为外部预警，不直接扣减资产；影响具体商品、类目、订单或结算时，转入 R07、R08、R15 或 R17。
- 建议动作：observe、cautious、manual_check。
- 核实事项：核实政策适用范围、平台规则版本、受影响商品和订单。

### R29 区域与物流环境风险

- 落地层级：二期或条件保留。
- 规则类型：辅助预警规则、结算延迟引用规则。
- 关联对象：区域、物流单、订单。
- 触发信号：自然灾害、区域管控、物流停运、仓库无法发货、重点区域拒收上升。
- 影响口径：外部自然灾害和区域监测不直接扣减资产。若已有物流单和订单数据证明配送延迟，转入 R12 或 R15；若导致退款、取消或赔付，转入 R16 或 R18。
- 建议动作：cautious、manual_check。
- 核实事项：核实受影响区域、订单、物流恢复时间、退款和拒收情况。

### R30 数据授权风险

- 落地层级：首期核心自动规则。
- 规则类型：数据治理规则、账户路径引用规则。
- 关联对象：API 授权、数据接口、店铺、回款账户。
- 触发信号：授权失效、接口延迟、字段缺失、数据断流、回款账户数据不可见。
- 影响口径：默认标记数据不足、无法估算原因和结论可信度下降；关键字段缺失时不得按正常口径估算资产。只有回款账户或代扣授权失效存在业务证据时，才转入 R20。
- 建议动作：manual_check；核心资产字段不可见且无法估算时可输出 pause_new 建议。
- 核实事项：核实授权状态、接口日志、缺失字段、数据快照时间和是否影响资产计算。

## 七、系统治理事件

系统治理事件用于运行排查、配置治理和审计留痕，不作为商户资产风险直接通知运营，也不参与可回款资产金额扣减。

### R31 规则与口径治理风险

- 落地层级：系统治理事件。
- 事件类型：数据治理、规则治理、配置治理。
- 关联对象：规则、资产口径、报告模板、Agent 提示词。
- 触发信号：规则误配置、资产口径版本冲突、误报率异常、关键规则未运行、报告模板缺失、提示词版本错误。
- 影响口径：不直接改变商家资产，但影响巡检结果可信度；报告必须标记规则或配置异常，不允许 Agent 猜测补全。
- 处理动作：manual_check、配置修正、版本回滚、重新运行巡检。
- 核实事项：核实规则版本、口径版本、提示词版本、命中日志、回测结果和发布记录。

### R32 内部操作治理风险

- 落地层级：系统治理事件。
- 事件类型：权限治理、配置治理、通知治理。
- 关联对象：用户、权限、配置、通知策略。
- 触发信号：人工改数、误配置通知对象、越权操作、重复发送、漏发、权限范围配置错误。
- 影响口径：不直接扣减资产；影响报告可信度、通知准确性和审计留痕。
- 处理动作：manual_check、权限修正、配置回滚、通知补发或撤回说明。
- 核实事项：核实操作日志、权限范围、配置变更、发布记录、通知记录和回滚方案。

## 八、平台规则依据和证据要求

涉及平台规则、公开说明或平台处罚依据时，必须记录：

1. 电商平台名称。
2. 规则名称。
3. 规则版本或发布日期。
4. 引用的业务含义。
5. 适用范围。
6. 数据来源或证据截图编号。

如果没有明确来源，不写成“平台公开规则规定”，只写成通用风险描述，例如“电商平台可能根据规则对违规商家采取扣罚、冻结、限制权限或清退等措施”。

每个自动规则至少保留：

1. 命中规则。
2. 关键指标。
3. 对比基准。
4. 数据时间。
5. 关联对象。
6. 影响金额口径。
7. 规则配置版本。
8. 资产口径版本。

缺少关键证据时，风险事件可以进入报告，但必须标记“证据不足”或“数据缺失”。如果规则引擎无法判断，应按配置输出兜底等级或“无法判断”状态，并说明原因。

## 九、平台适配附录

本文默认按通用电商平台口径编写。若首期明确以抖店为主要接入平台，以下字段按抖店口径理解，并映射到通用字段：

| 抖店口径 | 通用抽象字段 | 处理方式 |
| --- | --- | --- |
| 体验分 | 店铺服务能力评分 | 可用于 R04 辅助标记 |
| 精选联盟佣金 | 平台达人分销佣金 | 扣款统一进入 R18，达人依赖进入 R25 |
| 千川广告费 | 平台广告消耗 | 扣款统一进入 R18，投流效率进入 R27 |
| 抖店官方金融产品 | 平台官方金融扣款或平台内优先扣款 | 扣款统一进入 R18，权属/优先级冲突进入 R21 |

后续扩展到淘宝、京东、拼多多等其他平台时，应新增平台适配字段表，不改变通用风险主规则。

## 十、最终落地要求

1. 所有自动规则必须落到产品对象、规则类型、影响类型、建议动作和核实事项。
2. 所有金额必须来自规则引擎和资产口径配置，不能由文案或 Agent 自行计算。
3. 所有建议动作必须保持建议性质，不能写成系统已执行动作。
4. 重复主题只保留一个主规则，其他风险点只引用主规则，避免重复扣减。
5. 数据缺失、授权失效、接口延迟和口径冲突必须在报告中显式提示。
6. 首期默认按日巡检，订单级识别以数据可用性为前提。
7. 应收款存在不等于融资方能优先拿到钱，必须同时校验主体、账户、授权、权属和扣收优先级。
8. 辅助风险不直接扣减金额，必须转入主规则后才能影响可回款资产。
9. R31、R32 属于系统治理事件，不作为商户资产风险参与金额计算。
